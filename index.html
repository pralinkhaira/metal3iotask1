Our Aim

Metal3-io is an open source project that aims to provide a declarative approach and a toolset for managing bare metal infrastructure for bare metal using Kubernetes along with native API for managing bare metal hosts via a provisioning stack that is also running on Kubernetes. üòé

The project includes various components such as the Metal3-io operator, the Metal3-io bare metal host operator, the Metal3-io cluster-api provider, and the Metal3-io hardware manager.

Bare Metal Actuator

Bare Metal Actuator is an implementation of the Machine Actuator interface defined by the cluster-api project.This actuator reacts to changes to Machine objects and acts as a client of the BareMetalHost custom resources managed by the Bare Metal.

Bare Metal Operator

Metal3-io operator is responsible for managing the lifecycle of a bare metal host, which includes provisioning, configuring, and decommissioning. The operator uses the Kubernetes API to manage the state of the host.

Bare Metal Host Operator

Metal3-io bare metal host operator is responsible for discovering and managing the hardware resources of a bare metal host. It provides a framework for hardware management that allows users to interact with hardware resources using Kubernetes resources.

Cluster-api provider

The Metal3-io cluster-api provider is a Kubernetes cluster API implementation that allows the provisioning of Kubernetes clusters on bare metal infrastructure.

Bare Metal Hardware Manager

Metal3-io hardware manager provides a hardware inventory of a bare metal host, including information on the CPU, memory, storage, and network interfaces.

Our Mission

Our Mission is to make it more beginner friendly and easily discoverable and to improve the final user experience. We want to provide clear and concise information on what Metal3 is, how to use it and where to reach out for help in case of problems. Along with external resources links for additional information.The project aims to provide a unified and consistent management experience for both virtualized and bare metal environments, enabling users to treat their bare metal hosts as if they were just another Kubernetes resource.ü™õ
